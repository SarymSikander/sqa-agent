# Your Product — Technical Reference

This file provides additional context to the AI assistant beyond what's in the knowledge/ directory.

## How to use this file

Replace this file with a technical reference document for your own product. You can generate it by:

1. Reading your frontend repo (routes, components, forms)
2. Reading your backend repo (API endpoints, business logic, models)
3. Describing your user roles, portals, and key flows

## Recommended sections

- System Overview (what your product does, who uses it)
- Tech Stack
- User Roles and permissions
- Frontend Routes by portal/section
- Key User Flows (step by step)
- Backend API Endpoints
- Integrations (third-party services)

## Tips

- The more specific and accurate this file is, the better the AI assistant will answer questions about your product
- Focus on facts from the actual codebase, not aspirational documentation
- Keep it updated as your product evolves
